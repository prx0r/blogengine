# Source Atlas — Minimal Spec v0.1

## Core Principle

The system exists to turn source material into readable condensed essays and a growing concept glossary.

Keep it simple:

```text
Source Material → Condensed Essay → Concepts
```

Everything should trace back to source material.

---

# 1. Source Material

A source is anything uploaded or linked.

Examples:

* Corbin PDF
* Asprem academia paper
* Ficino chapter
* Steiner lecture
* Ibn ʿArabī passage
* Blog essay
* Transcript

## Minimal Fields

```json
{
  "id": "source_001",
  "title": "Mundus Imaginalis",
  "author": "Henry Corbin",
  "type": "pdf",
  "tradition": ["Islamic mysticism", "Sufism", "Iranian philosophy"],
  "source_url": "",
  "file_id": "",
  "raw_text": "",
  "clean_text": "",
  "notes": ""
}
```

## Rule

The author is just a field.

No separate author table at first.

Author pages can later be generated by grouping all source material with the same author name.

---

# 2. Essay

An essay is a readable output produced from one or more sources.

For now, there are only two essay types:

```text
condensed_source
bridge_essay
```

## condensed_source

A shorter version of a source text.

Mostly author wording, with AI notes only where needed.

## bridge_essay

A new essay that connects multiple sources/concepts.

More AI-written, but still grounded in source material.

## Minimal Fields

```json
{
  "id": "essay_001",
  "title": "Mundus Imaginalis — Condensed Reader",
  "type": "condensed_source",
  "source_ids": ["source_001"],
  "author": "Henry Corbin",
  "concepts": ["Imaginal World", "Active Imagination", "Symbol", "Theophany"],
  "prerequisites": ["Symbol", "Active Imagination"],
  "body": [],
  "notes": ""
}
```

## Essay Body Format

The body should be made from simple blocks.

```json
[
  {
    "kind": "ai",
    "text": "Corbin begins by correcting a modern misunderstanding of imagination."
  },
  {
    "kind": "source",
    "text": "The phrase mundus imaginalis is intended to designate a precise order of reality..."
  },
  {
    "kind": "ai",
    "text": "In plain terms, he is saying the imaginal is not fantasy. It is a real level of experience."
  },
  {
    "kind": "summary",
    "text": "The next section gives historical background and names several traditions that preserved this idea."
  }
]
```

## Body Block Types

Only use three:

```text
source = original author wording
ai = explanatory link or clarification
summary = compressed material from the source
```

That's enough.

No extra labels.

No complicated passage system yet.

---

# 3. Concept

A concept is a glossary node.

Concepts are also the essay tags.

No separate tag system.

## Minimal Fields

```json
{
  "id": "concept_imaginal_world",
  "name": "Imaginal World",
  "definition": "A real intermediate order of reality perceived through imagination, not fantasy.",
  "tradition": ["Sufism", "Iranian philosophy", "Corbin"],
  "synonyms": ["Mundus Imaginalis", "World of Image", "Intermediate World", "ʿĀlam al-Mithāl"],
  "related_to": ["Active Imagination", "Symbol", "Theophany", "Subtle Body"],
  "essays": ["essay_001"],
  "source_material": ["source_001"]
}
```

## Rule

Concepts should be specific.

Good:

```text
Daimon
Nous
Imaginal World
Active Imagination
Holy Guardian Angel
Kataphatic Imagination
Inner Senses
Subtle Body
Theophany
Symbol
```

Bad:

```text
metaphysics
mysticism
spirituality
religion
philosophy
history
esotericism
```

Those can be traditions or contexts, not concepts.

---

# 4. Tagging Rule

Essay tags are just concepts.

Each essay should have:

```text
3–5 concepts
```

A concept should be attached to an essay only if:

```text
The essay is actually about it.
```

Not just because it is mentioned once.

Example:

```json
{
  "essay": "Creative Imagination in Ibn ʿArabī",
  "concepts": [
    "Active Imagination",
    "Imaginal World",
    "Theophany",
    "Symbol",
    "Spiritual Perception"
  ]
}
```

---

# 5. Source Processing Flow

When a new source is uploaded:

```text
1. Save raw source.
2. Extract clean text.
3. Remove references, footnotes, page clutter, headers, and audio-awkward material.
4. Identify the main argument.
5. Create a condensed essay using source / ai / summary blocks.
6. Extract concepts.
7. Match concepts against existing glossary.
8. Add new concepts only if needed.
9. Attach 3–5 concept tags to the essay.
10. Link essay and source material back to those concepts.
```

---

# 6. Condensing Rule

The condenser should ask:

```text
How much can be removed while preserving the message?
```

Remove:

```text
references
bibliography
citation chains
academic padding
repetition
unnecessary historical detail
side arguments
audio-awkward phrasing
```

Keep:

```text
main thesis
definitions
key distinctions
important terms
strong author wording
central examples
conclusions
```

The output should read like:

```text
AI explains the move.
Author gives the key passage.
AI clarifies.
Author continues.
Low-signal section is compressed.
Author returns.
```

---

# 7. Difficulty

Do not use vague difficulty scores at first.

Use prerequisites.

```json
{
  "essay": "Mundus Imaginalis — Condensed Reader",
  "prerequisites": ["Symbol", "Active Imagination"]
}
```

Later, the system can turn prerequisites into levels like beginner/intermediate/advanced.

For now:

```text
difficulty = what concepts should be read first
```

---

# 8. Graph

The graph is generated from concepts.

Use only:

```text
Concept → related_to → Concept
Concept → essays → Essay
Concept → source_material → Source Material
Essay → concepts → Concept
Source Material → concepts → Concept
```

No complicated relationship types yet.

Just `related_to`.

If needed later, expand into:

```text
analogous_to
opposes
contains
accessed_by
```

But not now.

# 10. System Rule

Do not add new fields until the system actually needs them.

The first useful chain is:

```text
source uploaded
→ condensed essay created
→ concepts extracted
→ concepts linked to essay and source
→ concept page shows definition, related concepts, essays, and source material
```

That is enough.
