# Source Atlas — Minimal Spec v0.1

## Core Principle

The system exists to turn source material into readable condensed essays and a growing concept glossary.

```text
Source Material → Condensed Essay → Concepts
```

Everything should trace back to source material.

---

# 1. Source Material

A source is anything uploaded or linked.

Examples:

- Corbin PDF
- Asprem academia paper
- Ficino chapter
- Steiner lecture
- Ibn ʿArabī passage
- Blog essay
- Transcript

## Minimal Fields

```json
{
  "id": "source_001",
  "title": "Mundus Imaginalis",
  "author": "Henry Corbin",
  "type": "pdf",
  "tradition": ["Islamic mysticism", "Sufism", "Iranian philosophy"],
  "source_url": "",
  "file_id": "",
  "raw_text": "",
  "clean_text": "",
  "notes": ""
}
```

## Rule

The author is just a field. No separate author table. Author pages can later be generated by grouping all source material with the same author name.

---

# 2. Essay

An essay is a readable output produced from one or more sources.

Two essay types:

```text
condensed_source  — shorter version of a source, mostly author wording
bridge_essay      — connects multiple sources/concepts, more AI-written
```

## Minimal Fields

```json
{
  "id": "essay_001",
  "title": "Mundus Imaginalis — Condensed Reader",
  "type": "condensed_source",
  "source_ids": ["source_001"],
  "author": "Henry Corbin",
  "concepts": ["Imaginal World", "Active Imagination"],
  "prerequisites": ["Symbol", "Active Imagination"],
  "body": [],
  "notes": ""
}
```

## Essay Body Format

The body is an array of simple blocks:

```json
[
  { "kind": "ai", "text": "Corbin begins by correcting a modern misunderstanding of imagination." },
  { "kind": "source", "text": "The phrase mundus imaginalis is intended to designate a precise order of reality..." },
  { "kind": "ai", "text": "In plain terms, he is saying the imaginal is not fantasy." },
  { "kind": "summary", "text": "The next section covers historical background and related traditions." }
]
```

## Body Block Kinds (only three)

| kind | meaning | display colour |
|------|---------|----------------|
| `source` | original author wording | amber left-border + italic |
| `ai` | explanatory link or clarification | plain white text |
| `summary` | compressed material from source | muted/smaller grey text |

The colour differentiation helps readers immediately see which words are the author's and which are the AI's.

---

# 3. Concept

A concept is a glossary node. Concepts are also essay tags (no separate tag system).

## Minimal Fields

```json
{
  "id": "concept_imaginal_world",
  "name": "Imaginal World",
  "definition": "A real intermediate order of reality perceived through imagination.",
  "tradition": ["Sufism", "Iranian philosophy"],
  "synonyms": ["Mundus Imaginalis", "ʿĀlam al-Mithāl"],
  "related_to": ["Active Imagination", "Symbol", "Theophany"],
  "essays": ["essay_001"],
  "source_material": ["source_001"]
}
```

## Rule

Concepts should be specific. Good examples:

```text
Daimon | Nous | Imaginal World | Active Imagination
Holy Guardian Angel | Kataphatic Imagination | Inner Senses
Subtle Body | Theophany | Symbol
```

Bad (too vague — use as traditions/contexts instead):

```text
metaphysics | mysticism | spirituality | religion | philosophy | esotericism
```

---

# 4. Tagging Rule

Each essay should have 3–5 concepts. A concept should be attached only if the essay is actually about it, not just mentioned once.

---

# 5. Source Processing Flow

```text
1. Save raw source.
2. Extract clean text (remove refs, footnotes, page clutter, headers).
3. Identify the main argument.
4. Create a condensed essay using source/ai/summary blocks.
5. Extract concepts.
6. Match concepts against existing glossary.
7. Add new concepts only if needed.
8. Attach 3–5 concept tags to the essay.
9. Link essay and source back to those concepts.
```

---

# 6. Condensing Rule

Ask: how much can be removed while preserving the message?

Remove: references, bibliography, citation chains, academic padding, repetition, unnecessary historical detail, side arguments, audio-awkward phrasing.

Keep: main thesis, definitions, key distinctions, important terms, strong author wording, central examples, conclusions.

The output rhythm:

```text
AI explains the move → Author gives key passage → AI clarifies
→ Author continues → Low-signal section compressed → Author returns
```

---

# 7. Difficulty

Use prerequisites (concepts that should be read first), not vague difficulty scores.

```json
{ "prerequisites": ["Symbol", "Active Imagination"] }
```

---

# 8. Graph

Generated from concepts. Only these relations:

```text
Concept → related_to → Concept
Concept → essays → Essay
Concept → source_material → Source Material
Essay → concepts → Concept
Source Material → concepts → Concept
```

---

# 9. Database Shape (three tables)

```text
source_material  →  essay  →  concept
```

---

# 10. System Rule

Do not add fields until the system actually needs them. First useful chain:

```text
source uploaded
→ condensed essay created
→ concepts extracted
→ concepts linked to essay and source
→ concept page shows definition, related concepts, essays, and source material
```
