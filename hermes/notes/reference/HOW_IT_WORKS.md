# How The Whole System Actually Works

## The Core Loop (One Sentence)

Hermes runs every night, scans the knowledge graph for tensions, writes essays to resolve them, generates audio, and deploys — growing the system autonomously.

## The Three Pieces

### 1. sanskritree — What CAN be proved

Takes a claim. Checks if it can be formalized. If yes, tries to prove it in Lean. If no, marks it OUTSIDE_FORMAL or HOLLOW. The result is a **node graph** — some nodes are PROVED (geometry), some are OUTSIDE_FORMAL (Ficino's interpretations), some are HOLLOW (mystical claims). Each node tracks `reuse_count` — how many other claims depend on it. High reuse = genuinely central.

**Example flow for an astrology claim:**

```
Input: "Mars squares Saturn"
  → Sayability: formalizable (it's just geometry)
  → Library check: already proved? skip
  → Formalize: angular_distance(mars, saturn) = 90°
  → Prove via Pantograph: native_decide → PROVED
  → Store as node with reuse_count = 1

Input: "Mars square Saturn constrains spiritus"
  → Sayability: NOT formalizable (spiritus is an interpretive concept)
  → Mark: OUTSIDE_FORMAL
  → Source: Ficino, De Vita III.16
  → Store as node with provenance
```

### 2. Truthcores — What the system KNOWS

Each node in the graph IS a Truthcore. Every essay, every dialogue, every extracted claim becomes one. The fields:

```json
{
  "id": "tc_mars_saturn_square_001",
  "invariant": "Mars at 135° and Saturn at 225° are 90° apart",
  "lean_status": "PROVED",
  "mechanisms": [{"name": "Aspect Geometry", "desc": "90° square"}],
  "pedagogical_hooks": [
    {"target_state": "frustrated", "angle": "Structure is not your enemy"}
  ],
  "provenance": {"source": "caelus", "ref": "JPL DE440"},
  "reuse_count": 3
}
```

Every essay on the site generates Truthcores. Every HXRMXS dialogue generates them (the `n2_truthcore` block in every turn). Over time, high-reuse nodes emerge naturally.

### 3. Hermes — The AUTOMATION

Hermes runs three loops via cron:

**Daily (6 AM) — Consolidator**
- Scan new dialogues for Truthcores
- Run Lean verification on formalizable ones
- Update reuse_count
- Promote high-impact Truthcores

**Weekly (Sunday) — Gap Analyst**
- Find concepts with no good teaching pathway
- Find student states with no effective mechanism
- Generate Research Missions

**Nightly (2 AM) — Dreamer**
- Execute Research Missions (web search via Tool Gateway)
- For each tension (two Truthcores that contradict or don't compose):
  - Draft an essay outline
  - Research source material
  - Write the essay
  - Generate audio
  - Fetch matching art
  - Deploy to Cloudflare
  - Extract new Truthcores from the essay
  - Update the graph

## Concrete Example: What Happens When a User Asks Something

```
User (Telegram): "Saturn is retrograde and I feel stuck."

1. Hermes detects user state: "frustrated_block"
2. Queries Truthcore DB:
   - tc_saturn_retrograde_001 (PROVED: geometric proof)
   - tc_saturn_retrograde_meaning (OUTSIDE_FORMAL: sourced to Ficino)
3. Queries UNO graph for best teaching mechanism:
   - State "frustrated_block" + concept "saturn_retrograde"
   - Best mechanism: structural_analogy (success rate: 0.82)
4. Generates [PEDAGOGY] block via LoRA:
   student_state: frustrated_block
   mechanism: structural_analogy
   my_thoughts: "They're frustrated by constraint — use river metaphor"
5. Responds: "Saturn retrograde isn't a wall. It's like a river underground..."
6. Observes user's next turn → updates state
7. If prediction failed → logs for Dreamer consolidation
```

## The File You Need to Run

```bash
# 1. Install sanskritree
cd sanskritree
pip install -r requirements.txt

# 2. Run the proof engine on a claim
python run_proof_engine.py --claim "Mars squares Saturn"

# 3. It outputs:
#   Node 1: "Mars squares Saturn" — PROVED (geometry)
#   Node 2: "constrains spiritus" — OUTSIDE_FORMAL (Ficino)
#   Reuse count: 1

# 4. Hermes reads the node graph nightly
# 5. If reuse_count > threshold → it's a center node
# 6. If tension between nodes → Hermes writes an essay
```
