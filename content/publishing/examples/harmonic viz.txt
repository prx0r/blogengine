## 1. Background and Motivation

### 1.1 The Geometric Revolution in Music Theory

Dmitri Tymoczko's landmark work "A Geometry of Music" (2011) demonstrated that musical chords exist in non-Euclidean geometric spaces called orbifolds, where voice-leading relationships can be visualized as paths through multi-dimensional space. His OPTIC framework (Octave, Permutation, Transposition, Inversion, Cardinality) provides mathematical precision for analyzing harmonic relationships across musical styles from medieval polyphony to contemporary jazz.

However, three critical limitations persist:

1. **Visualization Challenge**: OPTIC spaces are abstract mathematical constructs. While theoretically elegant, they provide no intuitive visual representation of harmonic relationships.

2. **Computational Complexity**: OPTIC coordinates for complex chords require 58-dimensional vectors, creating memory bottlenecks for large-scale analysis.

3. **Static Geometry**: Current approaches treat chord progressions as discrete geometric movements without modeling the underlying forces that make certain transitions feel "natural" or "inevitable."

### 1.2 The Mathematical Opportunity

Recent advances in mathematical visualization and the availability of large-scale harmonic datasets create an unprecedented opportunity to address these limitations through novel mathematical applications:

**Gielis Curves**: Johan Gielis's generalization of circles to complex organic shapes provides a powerful visualization framework unexplored in music theory.

**Chebyshev Polynomials**: While extensively used in signal processing, these optimal approximation functions have never been applied to harmonic space compression.

**Boundary Value Problems**: Mathematical techniques for modeling physical phenomena offer new perspectives on harmonic relationships as energy fields.

**Chordonomicon Dataset**: Released October 2024, this dataset of 666,000 chord progressions with graph-based representations provides the scale necessary for deep learning validation.

### 1.3 Research Gap and Innovation

**Literature Review Findings**: Extensive search reveals **no prior applications** of Gielis curves, Chebyshev compression, or boundary value modeling to music theory. While L-systems have been used in algorithmic composition since the 1980s, their integration within a geometric-mathematical framework guided by the other novel techniques represents a new research direction.

**Innovation Positioning**: This research introduces genuinely novel mathematical tools to music theory while honestly integrating established techniques where appropriate.

## 3. Methodology

### 3.1 Mathematical Framework Development

#### 3.1.1 Gielis Chord Representation

**Innovation**: First application of Gielis curves to music theory.

**Approach**: 
- Map OPTIC coordinates to Gielis parameters: `m` (rotational symmetry) ← consonance measures, `n₃` (shape complexity) ← harmonic surprise
- Generate visual "fingerprints" for each chord type
- Create interactive visualization tools for harmonic exploration

**Expected Outcome**: Intuitive visual language for harmonic relationships.

#### 3.1.2 Chebyshev Feature Compression

**Innovation**: Novel application of optimal approximation theory to harmonic space.

**Approach**:
- Approximate 58D OPTIC vectors using truncated Chebyshev expansions
- Optimize polynomial orders for memory efficiency vs. information preservation
- Validate compression quality through reconstruction error analysis

**Expected Outcome**: 5-10x memory reduction while preserving geometric relationships.

#### 3.1.3 Boundary Value Harmonic Modeling

**Innovation**: First physics-based model of chord transitions.

**Approach**:
- Formulate Laplace equations on Gielis-defined harmonic domains
- Model voice-leading distances as boundary conditions
- Solve for harmonic potential fields governing chord transitions

**Expected Outcome**: Universal mathematical laws underlying voice-leading preferences.

#### 3.1.4 L-Systems Integration

**Established Technique**: Honest acknowledgment of extensive prior work since 1980s.

**Novel Application**: Integration within geometric-mathematical constraint framework.

**Approach**:
- Use L-systems for hierarchical pattern generation
- Constrain production rules by Gielis geometry and boundary value solutions
- Generate fractal-like harmonic structures following mathematical principles

### 3.2 Computational Architecture

#### 3.2.1 Memory-Efficient Graph Neural Networks

**Architecture**: Custom geometric GNN optimized for 6GB VRAM constraints.

**Key Features**:
- Gradient accumulation (effective batch size 32 with physical batch size 4)
- Mixed precision training (bf16)
- Chebyshev-compressed node features
- Graph sampling for large-scale training

**Training Strategy**:
- Stage 1: Train on Gielis-encoded chord representations
- Stage 2: Add Chebyshev compression optimization
- Stage 3: Integrate boundary value constraints
- Stage 4: Full model with L-system generation

#### 3.2.2 Dataset Integration

**Primary Dataset**: Chordonomicon (666,000 chord progressions with graph structure)

**Preprocessing Pipeline**:
1. Convert Harte chord notation to OPTIC coordinates
2. Generate Gielis curve parameters for visualization
3. Compute Chebyshev approximations for compression
4. Solve boundary value problems for transition potentials
5. Create graph representations for GNN training
